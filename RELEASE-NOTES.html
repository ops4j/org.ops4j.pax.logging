Release Notes - Pax Logging - Version 1.2.0

For this project, compatibility is defined as;

  • No changes to method signatures of any of the classes in the API. Only new methods may appear. Note that this may break compatibility with additional Pax Logging implementations, but that is outside the compatibility scope.
  • The configuration system used, whereby a Configuration object either contains the URL or file reference to the Log4j configuration, OR that the Configuration instance itself contains the Log4j configuration in its properties format.
  • Pax Logging version 1.x will depend on JDK1.4
  • That we are and will remain compatible with the following Legacy APIs, to the greatest extent possible;
    • Apache Log4j, version 1.2.15
    • Apache Jakarta Commons Logging, version 1.1
    • Apache Avalon Logging API, version 1.0
    • JDK Logging, version 1.4
    • Knopflerfish Log, version 1.3
    • OSGi Log Service, version 1.3
    • Simple Logging Facade for Java (SLF4J), version 1.4.3
    • Pax Logger API, ver 1.0
